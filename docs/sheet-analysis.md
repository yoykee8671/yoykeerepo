# WooofPay 시트 구조 분석

분석 기준 문서: `2026_선매입 브랜드 관리대장`

## 확인된 탭 구조

- 기본 집계 탭: `입금요청시트`
- 브랜드/업체 운영 탭: `테일릿(주미스)`, `펫페이스`, `니드포펫`, `릴리스키친`, `페슬러`, `아롬나옴`, `몽슈슈`, `로렌츠(B2B)`, `카네브`, `봉쥬르뚜뚜`, `퍼펫`, `레인보우스토리`, `오고니아`, `안스펫`, `헤이마`, `누핀`, `아카바코퍼레이션 (패디펫)`, `페치 (테일스키친)`, `퍼피갤러리`, `미티본(펫대디)`, `오느루`, `포포네`, `주로(에이파크)`, `쉬오트`, `인히어런트`, `어니스트밀`, `스윙도츠`, `골드로니`, `펫츠그린`, `플러쉬퍼피`, `앤블랭크`, `뮤니쿤트`, `누누숨`, `고공캣`, `클러스터라운드`, `아인솝`, `분독`, `리케이(아이그룸)`, `펫에스테(GLC)`, `베럴즈`, `트러스티푸드(림피드)`, `온힐`, `브릿지독`, `리꼬르소`, `포사이어티 (시카로 / 논스톱)`, `지노네이처 (포엣미)`, `빌리스벳`, `이비야야(도기파크)`, `위러브코코`, `복슬강아지`, `스쿱543`, `쿠루름`, `룰루키친`, `테일하이`, `꼬뜨cote`, `닥터웰릿(곰곰연구소)`, `콘디(삼보첨단)`, `프롬한라(벨아벨팜)`, `리카리카`
- 참고/단가표 탭: 이름에 `단가표`, `양식`, `템플릿`, `행사공급가`가 포함된 탭

## 입금요청시트 컬럼

| 기존 컬럼 | 웹앱 필드 | 용도 |
| --- | --- | --- |
| 브랜드명 | `brandName`, `brandId` | 요청을 업체와 연결 |
| 주문번호 | `orderNo` | 주문 식별자 |
| 주문자명 | `customerName` | 고객 식별 |
| 업체 실 입금액 | `depositAmount` | 지급 요청 금액 |
| 입금(예정)일자 | `expectedDepositDate` | 예정 또는 실제 처리 기준일 |
| 출고마감시간 | `cutoffNote` | 출고마감, 위탁정산, 송장입력 후 지급 등 업체별 운영 메모 |
| 원본 시트 | `sourceSheet` | 요청이 나온 브랜드 탭 |
| 행 번호 | `sourceRow` | 원본 행 추적 |
| 필수 메모 및 계좌번호 확인 | `requiredMemo` | B2B, 계좌 확인, 기타 주의사항 |
| 사업자명 | `businessName` | 업체 정산 정보 |
| 사업자번호 | `businessNumber` | 업체 정산 정보 |
| 입금자명 | `depositorName` | 입금 표시명 |

## 웹앱으로 옮긴 규칙

- 브랜드 기본 정보와 입금요청 행을 분리해서 저장한다.
- 기존 시트의 `브랜드명 + 주문번호 + 원본 시트 + 행 번호` 조합은 추적 키로 취급한다.
- 사업자명, 사업자번호, 입금자명, 출고마감/정산 메모는 브랜드 기본값으로 보관하고 요청별로 덮어쓸 수 있다.
- 브랜드별 정산 유형은 `선매입-채권`, `선매입-일반(수수료)`, `선매입-일반(공급가)`, `위탁`으로 관리한다.
- 공급가형 브랜드는 브랜드별 품목 단가 마스터를 두고, 품목별 `적용 시작일` 기준 최신 단가를 사용한다.
- 단가 개정은 이력 행으로 누적 저장하며, 입금요청에는 실제 사용된 품목/수량/적용 단가를 함께 저장한다.
- 위탁 건은 `위탁-입금전` 상태로 별도 필터링하고 실시간 대기금액에서 제외한다.
- 채권 브랜드는 총 채권액과 누적 차감액을 함께 관리한다.
- 배송비는 브랜드 규정을 텍스트로 보관하고, 주문건별 실제 배송비는 요청 입력 시 별도로 반영한다.
- 제품매출은 배송비를 포함하지 않으며 수수료 계산은 제품매출 기준으로만 수행한다.
- 관리자 계정의 생성, 수정, 삭제와 입금요청/브랜드 변경은 모두 감사 로그에 기록한다.
- 브랜드별 공유 링크는 읽기 전용이다.
- Excel 추출은 브랜드별 또는 전체 입금요청 단위로 제공한다.
- Google Sheets 아카이브는 웹앱 데이터가 원본이고, Apps Script Webhook을 설정하면 시트로 동기화한다.

## 보류한 항목

- ChatGPT 프로젝트 히스토리 링크는 외부 접근 권한이 없어 직접 읽지 못했다.
- Google Sheets 쓰기 권한은 서비스 계정 또는 Apps Script 배포 URL이 필요하므로, 현재 앱은 Webhook URL이 설정된 경우에만 실제 시트 쓰기를 수행한다.
